Step-3

The endpoint is:

bash

POST /prweb/api/v1/pipelines

You must send a JSON payload describing the pipeline.
Usually, you’ll:

First get a template JSON by calling GET /pipelines/{id} on an existing pipeline.

Modify it (name, stages, environments, product details, etc.).

Post it to create the new pipeline.




Example JSON Payload
Here’s a minimal structure (you’ll need to adjust values to match your environments and product rulesets in Pega):


{
  "name": "My New Pipeline",
  "application": "MyApp",
  "applicationVersion": "01.01.01",
  "branch": "main",
  "stages": [
    {
      "name": "Dev",
      "tasks": [
        {
          "taskName": "Deploy to Dev",
          "taskType": "DEPLOY"
        }
      ]
    },
    {
      "name": "QA",
      "tasks": [
        {
          "taskName": "Deploy to QA",
          "taskType": "DEPLOY"
        }
      ]
    }
  ]
}
